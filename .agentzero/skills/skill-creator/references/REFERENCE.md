# Skill Creator Reference

## Naming Rules
- 1 to 64 characters
- lowercase letters (`a-z`), digits (`0-9`), and hyphens only
- cannot start or end with a hyphen
- cannot contain consecutive hyphens
- must match the parent directory name

## Minimum Skill Shape
A skill directory should contain at least:

- `SKILL.md` (required)

Optional:

- `scripts/`
- `references/`
- `assets/`

## Description Guidance
A strong description should say:
- what the skill does
- when to use it

Example:
`Extracts entities from invoices and validates totals. Use when processing invoice PDFs or images.`
